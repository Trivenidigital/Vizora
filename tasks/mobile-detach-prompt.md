# Detach Vizora Mobile Companion App into Standalone Repository

The Vizora mobile companion app (`mobile/`) currently lives inside the main Vizora monorepo. It needs to be extracted into its own standalone repository so it can be independently built, tested, and shipped via Expo EAS to iOS App Store and Google Play.

**What this app IS:** An Expo/React Native management app for business users to log in, view displays, scan QR codes to pair devices, upload content, push content to displays, and manage playlists — all from their phone.

**What this app is NOT:** A display client. It does not render templates or show content on a screen. That's the Android TV app (`vizora-tv` repo) and the Electron desktop app (`display/`).

This is a TWO-PHASE project. Do NOT make changes until Phase 1 is reviewed.

## PHASE 1: Audit & Extraction Plan

### Step 1: Verify App Architecture

The app is:
- **Framework:** Expo SDK 54 + React Native 0.81 + React 19 + TypeScript
- **Router:** expo-router 6 (file-based routing in `app/`)
- **State:** Zustand 5 stores (`src/stores/`)
- **Bundle ID:** `com.vizora.companion` (both iOS and Android)
- **Native dirs:** Gitignored (`/ios`, `/android`) — generated by `expo prebuild`
- **Build system:** EAS Build (`eas.json` with development/preview/production profiles)
- **Tests:** Jest 30 + jest-expo + @testing-library/react-native
- **E2E:** Maestro flows (`.maestro/`)
- **Package manager:** npm (has `package-lock.json`, NOT in pnpm workspace dependency graph)

Verify:
```
- No imports from @vizora/database, @vizora/middleware, or other workspace packages
- No shared code with vizora-tv (Android TV display client) — completely different purpose
- Only monorepo coupling is the entry in pnpm-workspace.yaml
- App has its own node_modules (npm, not pnpm)
```

### Step 4: Document Communication Protocol

The mobile app talks to the same backend as the web dashboard:

```
AUTH (user JWT, NOT device JWT):
- POST /api/v1/auth/login        → { email, password } → { access_token, user }
- POST /api/v1/auth/register     → { email, password, firstName, lastName, organizationName }
- GET  /api/v1/auth/me           → current user info

DISPLAYS:
- GET    /api/v1/displays              → list org displays
- GET    /api/v1/displays/:id          → display detail
- PATCH  /api/v1/displays/:id          → update nickname/location/playlist
- DELETE /api/v1/displays/:id          → remove display
- POST   /api/v1/displays/:id/push-content    → push content to display
- POST   /api/v1/displays/:id/screenshot      → request screenshot
- GET    /api/v1/displays/:id/screenshot       → get screenshot URL

CONTENT:
- GET    /api/v1/content               → list content (with filters)
- GET    /api/v1/content/:id           → content detail
- POST   /api/v1/content/upload        → multipart upload (XHR for progress)
- DELETE /api/v1/content/:id           → delete content

PLAYLISTS:
- GET    /api/v1/playlists             → list playlists
- GET    /api/v1/playlists/:id         → playlist detail
- POST   /api/v1/playlists/:id/items   → add item
- DELETE /api/v1/playlists/:id/items/:itemId → remove item
- POST   /api/v1/playlists/:id/reorder → reorder items

PAIRING (user-side — scans QR, sends code):
- POST /api/v1/devices/pairing/complete → { code, nickname? } → { display }

WEBSOCKET (realtime status updates via Socket.IO):
- Connects to EXPO_PUBLIC_REALTIME_URL with Bearer token
- Receives: device:status (online/offline), screenshot:ready
- Same Socket.IO protocol as web dashboard (NOT the device protocol)

RESPONSE ENVELOPE:
- All API responses wrapped in { success, data, meta }
- Client unwraps automatically in api/client.ts
```

## PHASE 2: Extraction

### Step 1: Create Standalone Repository

```bash
# Create GitHub repo
gh repo create Trivenidigital/vizora-mobile --private --description "Vizora Companion — mobile app for managing digital signage displays"

# Create local directory
mkdir ../vizora-mobile
cd ../vizora-mobile
git init

# Copy files (excluding node_modules)
cd ../vizora
tar cf - --exclude='mobile/node_modules' --exclude='mobile/.expo' mobile/ | (cd ../vizora-mobile && tar xf - --strip-components=1)

# Initial commit
cd ../vizora-mobile
git add -A
git commit -m "chore: initial commit — extract from Vizora monorepo"
git remote add origin https://github.com/Trivenidigital/vizora-mobile.git
git branch -M main
git push -u origin main
```

### Step 2: Verify Standalone Build

```bash
cd ../vizora-mobile
npm install

# Verify tests pass
npx jest

# Verify Expo config is valid
npx expo config --type public

# Verify TypeScript compiles
npx tsc --noEmit
```

## HARD RULES

- This is a COMPANION app (management), NOT a display client
- Zero shared code with vizora-tv (Android TV display app) — different purpose entirely
- Auth is USER JWT (email/password login), not DEVICE JWT
- Bundle ID: `com.vizora.companion` — do NOT change
- Keep npm (not pnpm) — Expo projects use npm by convention
- No native ios/ or android/ dirs in git — they're generated by `expo prebuild`
- EAS Build config (`eas.json`) must remain functional
- Maestro E2E flows (`.maestro/`) must be preserved
- Backend stays in monorepo — this app is just a REST/WebSocket client
- The monorepo must work with ZERO dependencies on the mobile app
