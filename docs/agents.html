Request Analyzer request-analyzer
Classifies raw support requests into structured JSON: { type, component, urgency, description }. Supports 6 request types (bug-fix, feature-request, config-change, ui-tweak, content-update, unknown) across 10 domain components. Falls back to unknown if confidence <60%.

Code Scout code-scout
Receives classification JSON and searches domain-specific paths to find up to 15 relevant files with targeted line ranges. Follows imports 1 level deep and always includes associated test files. Prioritizes: entry points > services > hooks > tests > config.
.claude/agents/code-scout.md

Plan Writer plan-writer
Takes classification + file list and generates a structured implementation plan: summary, files to modify (with line ranges), files to create, affected tests, risk assessment (Low/Medium/High), and estimated scope. Describes changes in prose — never writes code. Capped at 50–100 lines.

Risk Assessment Criteria
Low — single file, cosmetic/text, no logic change, existing tests cover it
Medium — 2–5 files, logic changes, may need new tests, no auth/data impact
High — 6+ files, auth/security/data migration, infrastructure, breaking API
Plan Writer Output Structure
1. Summary — what changes and why
2. Classification — type, component, urgency
3. Files to Modify — with specific line ranges
4. Files to Create — if needed
5. Tests — existing + new tests required
6. Risk Assessment — level, justification, rollback
7. Estimated Scope — files, lines, complexity
Red Flags — Agent Stops and Asks
Request touches auth or data migrations · Classification is unknown · Plan affects 10+ files · Verification fails after implementation
