# Mobile Companion App Detach — Phase 1 Audit

## App Identity
- **Name:** Vizora Companion (`@vizora/mobile`)
- **Type:** Management app (NOT a display client)
- **Framework:** Expo SDK 54 + React Native 0.81 + React 19 + TypeScript
- **Bundle ID:** `com.vizora.companion` (iOS + Android)
- **Build system:** EAS Build (development/preview/production profiles)
- **Package manager:** npm (own package-lock.json)

## File Inventory
- **57 source files** (app/, src/, .maestro/, assets/)
- **12 config files** (package.json, tsconfig.json, app.json, app.config.ts, eas.json, jest.config.ts, jest.setup.ts, .gitignore, etc.)
- **No native dirs in git** — ios/ and android/ are gitignored (generated by `expo prebuild`)

## Monorepo Coupling: ZERO
- [x] No `@vizora/*` dependencies in package.json
- [x] All imports are relative (`../../src/...`) — no cross-package refs
- [x] tsconfig.json has no path aliases pointing outside mobile/
- [x] No shared code with vizora-tv or display/
- [x] app.config.ts uses EXPO_PUBLIC_* env vars only
- [x] eas.json is self-contained
- [x] Only coupling: one line in `pnpm-workspace.yaml` (`- mobile`)

## Communication Protocol
- **Auth:** User JWT (email/password), Bearer token in Authorization header
- **API:** REST calls to /api/v1/* (auth, displays, content, playlists, pairing)
- **Realtime:** Socket.IO to realtime gateway (device status updates)
- **Response format:** `{ success, data, meta }` envelope (auto-unwrapped by client)
