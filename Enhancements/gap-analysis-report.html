11. Recommended Implementation Roadmap

Strategy: Phase 1 closes critical gaps and quick wins. Phase 2 achieves competitive parity with OptiSigns Standard tier. Phase 3 targets differentiation. Phase 4 is enterprise/premium features.

Phase 1: Foundation & Quick Wins (Weeks 1-3)

Quick Wins (1-2 days each)

1. Screen orientation enum expansion (landscape → 0/90/180/270)

2. Display tag API endpoints (schema already exists)

3. Playlist metadata computed fields (totalSize, totalDuration)

4. Template orientation metadata field

5. Playlist duplicate endpoint

6. Schedule duplicate endpoint

Foundation Work (1-2 weeks)

7. MinIO/S3 integration for file storage (critical for multi-instance production)

8. Analytics query API from ClickHouse (content impressions already being stored)

9. Role-based access audit — apply @Roles guards consistently

Phase 2: Competitive Parity with OptiSigns Standard (Weeks 4-10)

UI/UX Enhancements

10. Three-panel drag-and-drop playlist builder

11. Calendar-based schedule view (FullCalendar.js integration)

12. Playlist preview player component

13. Content grid/list view toggle with folder organization

14. Device search, sort, filter controls + bulk operations

Analytics & Monitoring

15. Proof of Play dashboard with time-series charts

16. Device offline alert notifications (email/webhook)

17. Device uptime analytics from heartbeat history

Team & Account

18. Team member invitation and management UI

19. Audit log viewer page

20. API key management system

Phase 3: Differentiation & Growth (Weeks 11-20)

Template Ecosystem

21. Template library with searchable catalog (seed 50-100 templates)

22. Visual template editor (evaluate GrapesJS, Craft.js, or Polotno)

23. AI template generation via Claude/GPT API

Integrations

24. Widget architecture with standardized data source adapters

25. Weather, RSS, clock, countdown — initial widget set

26. Split-screen / multi-zone layout system

27. QR code overlay generation and placement

Device

28. Remote device screenshot/preview via WebSocket command

29. Operational schedule (auto on/off) per device

Phase 4: Enterprise & Premium (Weeks 20+)

Enterprise Features

30. Stripe billing integration (subscription tiers, payment, invoicing)

31. SAML SSO via passport-saml

32. Content approval workflow (draft → review → approved)

33. Custom branding / white-label support

34. Campaign management (group schedules + content + displays)

35. Google Workspace / MS365 integrations

12. Vizora Competitive Strengths (Keep & Promote)

These are areas where Vizora already matches or exceeds OptiSigns. These should be highlighted in marketing and preserved during feature development.
StrengthDetailOptiSigns Comparison
Real-time WebSocket Architecture Socket.IO with device/org rooms, dual Redis+Postgres persistence, 15s heartbeat, content impressions, pending commands OptiSigns uses generic cloud push — Vizora's architecture is more sophisticated and responsive
REST API with Swagger Docs (All Plans) Full API documentation at /api/docs, all endpoints accessible programmatically OptiSigns restricts GraphQL API to Enterprise tier ($45/screen/mo). Vizora offers API to all users.
Security Stack Dual JWT (user+device), httpOnly cookies, CSRF, Helmet, XSS sanitization, 3-tier rate limiting, bcrypt-14, SHA-256 token hashing More layers of defense than visible in OptiSigns. Token hashing in DB is notable.
Content Expiration with Auto-Replacement Hourly cron expires content, auto-replaces with designated replacement, auto-removes from playlists Not prominently featured in OptiSigns. Valuable for time-sensitive promotions.
File Versioning with Rollback Replace file with backup, version chain via previousVersionId, full history API Not visible in OptiSigns. Prevents accidental content loss.
Template Data Source Security (SSRF Protection) Private IP blocking, cloud metadata blocking, 10s timeout, circuit breaker Shows enterprise-grade security thinking even in template data fetching.
Self-Hosted Deployment Option PM2 cluster mode, Docker infrastructure, no vendor lock-in OptiSigns is cloud-only. Self-hosted is valuable for regulated industries.
Multi-Tenant Architecture Organization-based isolation at DB level, all queries scoped, no cross-org leakage Comparable to OptiSigns sub-accounts, but Vizora's is enforced at the database query level.
